Test driven development for data scientists

Why? Because way too many folks in data science have not been exposed to good product development practices (XP + BDD + CI/CD). I did a few trial iterations with small classes of 4-5 people. Feedback is good, but 2 days is not enough. Needs some improvements with a common setup, API testing, mocking, and ML specific testing. Current version is 3 days or 2.5 days with some content skipped.
